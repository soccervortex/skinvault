Onderwerp: Volledige migratie van Upstash KV naar Multi-Cluster MongoDB Backup Systeem

"Ik krijg momenteel een UpstashError: max requests limit exceeded. Ik wil Upstash KV volledig verwijderen uit de applicatie en alle functionaliteit die voorheen door KV werd afgehandeld (zoals caching, rate-limiting of tijdelijke data) nu direct via MongoDB laten lopen.

Belangrijke vereisten voor de implementatie:

Verwijder KV: Zoek alle imports van @upstash/redis of @upstash/kv en verwijder deze. Vervang de logica door MongoDB operaties.

Multi-Cluster Backup Logica: Behoud en verbeter de huidige 'failover' logica. De applicatie moet de onderstaande MongoDB clusters gebruiken. Als Cluster 1 vol is of een limiet bereikt, moet hij overstappen naar Cluster 2, enzovoort.

Cluster 1: mongodb+srv://skinvaults:System1153@cluster0.5ceoi.mongodb.net/?appName=Cluster0

Cluster 2: mongodb+srv://skinvaults:System1153@cluster0.09vuhs5.mongodb.net/?appName=Cluster0

Cluster 3: mongodb+srv://skinvaults:System1153@cluster0.n03bi8z.mongodb.net/?appName=Cluster0

Cluster 4: mongodb+srv://skinvaults:System1153@cluster0.kcz6p8s.mongodb.net/?appName=Cluster0

Cluster 5: mongodb+srv://skinvaults:System1153@cluster0.edzgnci.mongodb.net/?appName=Cluster0

Data Consistentie & Migratie: Zorg ervoor dat bij het schrijven naar een nieuw cluster, de noodzakelijke gegevens overgezet worden of dat de app weet in welk cluster de data van deze maand staat.

State Management: Implementeer een robuuste check (bijvoorbeeld op basis van de huidige datum of een 'connection test') om te bepalen welk cluster momenteel actief moet zijn.

Volledige Bestanden: Pas niet alleen de database connectie aan, maar herschrijf de volledige functies in de API routes en lib-files zodat de overgang naadloos is zonder KV-verwijzingen.

Pas dit toe op alle relevante bestanden (zoals database config, middleware, en API endpoints waar de fout optrad)."

Aanvullende tips voor je setup:
Database Utility: Vraag Cursor om een db.js of mongodb.ts utility file te maken die de getConnectedClient() logica afhandelt. Hierin kan hij de logica stoppen die checkt: "Is het cluster vol? -> Gebruik de volgende".

Collections: Omdat KV vaak voor Key-Value opslag wordt gebruikt, is het handig als Cursor een specifieke MongoDB collection aanmaakt genaamd kv_replacement of cache om de oude KV-structuur te simuleren met een key en een value veld.
